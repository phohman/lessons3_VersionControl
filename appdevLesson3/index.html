Highlight #1 Version Control – Summary

Version Control is one main place for all project files. Team members check files out make changes and check them back in. All versions of a file are saved. This system keeps a revision history including when a file was changed, who changed it, how the versions differ and it merges changes to a file. The advantages of a VCS are that files cannot be overwritten, people can work on the same file simultaneously and you can go back to previous versions. Each system has a remote repository that is shared by multiple people and a local repository that is on the individual’s computer.

Highlight #2 Types of Version Control Systems

There are three main types of Version Control Systems: Local, Centralized and Distributed. One common local system is called RCS, it keeps track of the differences between files and there is only one system so if it is corrupted data can be lost. A Centralized system allows people to collaborate by using a single server that has all version of a file and it gives administrators full control. One disadvantage of a centralized system is that if the server goes down work stops. A distributed system like Git provides a full copy of the data. If one copy goes down or is corrupted there is a backup available.

Highlight #3 Git Basics

Git is an open source distributed version control system that can handle small or large projects. It was created in 2005 and is fast, simple, non-linear. A full copy of the information is stored locally on locally on a machine and a copy is on a main server. A local repository allows a person to be able to work off line and it makes the system fast. Git saves changes with a snapshot or picture of how the data looks at a specific time. Each file can either be tracked or untracked. Tracked files can be unmodified, modified, or staged. There are three sections of a git project, the git directory, the working directory and the staging area.

Highlight #4 Git Commands.

There are three steps in Git - Add, Commit, and Push. To begin you can either import or clone a Git repository. $ git init – creates a new local subdirectory that contains all needed files $ git add *.c $git add LICENSE $git commit -m “initial project version’ Or $git clone “web address” – this create a directory and initializes it. $ git status – view all of the new or modified files to be committed.
